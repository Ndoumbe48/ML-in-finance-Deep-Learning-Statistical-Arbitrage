# ML-in-Finance-Deep-Learning-Statistical-Arbitrage

A reproduction of the **Deep Learning Statistical Arbitrage** paper, adapted for cryptocurrency markets.

---

## Project Overview

This project implements deep learning-based statistical arbitrage strategies on cryptocurrency price data. It includes a complete pipeline for data collection, feature extraction via PCA, and training of trading policy models.

---

## Directory Structure

### Data Collection

**`data.py`**
- Fetches OHLCV (Open, High, Low, Close, Volume) data from Binance API
- Retrieves circulating supply data from CoinGecko API
- Generates two main datasets:
  - `crypto_prices_4h_2022_2026.csv`: 4-hourly price data for all cryptocurrencies
  - `crypto_top40_by_marketcap_2022_2026.csv`: Top 40 cryptocurrencies by market cap at each 4-hour timestamp

### Residual

**`factor_model/pca.py`**
- Implements Principal Component Analysis (PCA) on price returns
- Extracts principal components as trading factors
- Generates residual time series for different numbers of factors
- Outputs residual data arrays for use in trading models

### Data Storage

**`residuals/`**
- Contains residual time series datasets generated by `pca.py`
- These residuals serve as inputs to the trading policy models

### Configuration

**`config/`**
- Contains YAML configuration files for different trading strategy tests
- Defines model parameters, hyperparameters, and training/testing specifications
- Each config file specifies a unique trading policy experiment

### Models

**`models/`**
- Contains implementations of trading policy models
- Includes neural network architectures and other machine learning models
- Defines the decision-making logic for trading signals

### Training & Backtesting

**`train_test.py`**
- Core training and simulation engine
- Trains trading policy models on historical data
- Simulates trading execution and calculates performance metrics
- Handles data preprocessing, model fitting, and results computation

**`run_train_test.py`**
- User-friendly interface to `train_test.py`
- Manages configuration loading, logging, and results saving
- Handles command-line arguments for easy experimentation

### Utilities

**`utils.py`**
- Provides helper functions used throughout the project

---


### Prerequisites

```bash
pip install -r requirements.txt
```

### Running a Trading Strategy

To train and backtest a trading policy:

```bash
python3 run_train_test.py -c configs/config_name_here.yaml
```
